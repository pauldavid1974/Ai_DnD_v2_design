# Architecture Blueprint & Tech Stack {#architecture-blueprint-tech-stack}

**CRITICAL DIRECTIVE:** This document defines the structural boundaries of the Android application. Deviation from these architectural layers is strictly forbidden. The system is built on a \"Software 2.0\" paradigm, enforcing an absolute decoupling between local deterministic logic and remote probabilistic generation.

## 1. Core Architecture Pattern {#core-architecture-pattern}

The application follows **Clean Architecture** combined with **MVVM (Model-View-ViewModel)** and **MVI (Model-View-Intent)** principles to ensure strict Unidirectional Data Flow (UDF).

- **Presentation Layer (UI):** Built entirely in Jetpack Compose (Material 3). Observes immutable StateFlow from the ViewModel. Triggers actions exclusively via typed Intents.

- **Domain Layer:** Pure Kotlin. Contains Use Cases, the Deterministic Rules Engine (\"The Bones\"), and repository interfaces. This layer has ZERO knowledge of the UI, Room, or Network frameworks.

- **Data Layer:** Implements repository interfaces. Manages Room SQLite (local state/SRD) and Ktor (remote LLM API).

## 2. Dependency Injection {#dependency-injection}

- **Framework:** Dagger Hilt.

- **Rule:** All ViewModels, Use Cases, Repositories, and Network clients MUST be injected via Hilt modules. No manual instantiation or Singleton object abuse for core components.

## 3. Database Layer (Room SQLite) {#database-layer-room-sqlite}

- **Role:** The ultimate source of truth for both immutable game rules and mutable player state.

- **Immutable Data (SRD):** Reference tables for spells, classes, items, feats, and monsters.

- **Mutable Data (Campaign):** Character sheets, inventory, active HP, and the memory_updates logs generated by the background Chronicler.

- **Constraint:** The LLM API NEVER executes writes to this database. Only the local Domain layer updates state after mathematical adjudication.

## 4. Network Layer (Ktor Client) {#network-layer-ktor-client}

- **Role:** Communication with \"The Brain\" (LLM Providers like Anthropic, OpenAI, Groq).

- **Implementation:** Ktor Client with SSE (Server-Sent Events) support for real-time typewriter text streaming.

- **Security:** API keys are accessed via Bring-Your-Own-Key (BYOK) architecture using Android\'s EncryptedSharedPreferences. Keys must never be logged or stored in plain text.

## 5. The Neuro-Symbolic Decoupling {#the-neuro-symbolic-decoupling}

The most critical architectural rule: The local engine logic (math, dice, HP tracking) and the network logic (LLM API calls) must NEVER directly invoke each other. They communicate exclusively through the Domain layer via strongly-typed Kotlin Data Classes (@Serializable), which represent the abstracted JSON contracts.
